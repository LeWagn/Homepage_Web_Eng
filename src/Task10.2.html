Fehler finden mit Typescript

Fehlerhaftes Code-Beispiel:

const add = (x,y) => x+y;
const equals = (x,y) => x===y;

console.log( add(1,2) );
console.log( add(true, true ) );
console.log( add(true, false ) );
var x = y = 1;
console.log( add(equals(x,y), equals(y,x)) );

Übersetzung in ein (fehlerhaftes) TypeScript Programm:

const add = (x: number, y: number): number => x + y;
const equals = (x: any, y: any): boolean => x === y;

console.log(add(1, 2));
console.log(add(true, true));
console.log(add(true, false));
let x: number = 1;
let y: number = 1;
console.log(add(equals(x, y), equals(y, x)));

TypeScript akzeptiert in der Funktion add()die numerischen und booleschen Werte nicht. Dies würde zu einen Fehler beim Compilieren führen.
Weiterhin können blocked-scope Variablen (wie in Z.6) nicht neu deklariert werden.

Wie kann TypeScript sonst noch bei der Fehlersuche helfen?

Weitere Fehlerhilfe durch TypeScript:

  • Typfehler: Wenn ein Wert einen falschen Typ hat, z.B. eine Zahl statt eines Strings, wird ein Typfehler gemeldet.
  • Variablenfehler: Wenn eine Variable nicht richtig deklariert oder verwendet wurde, wie z.B. eine nicht initialisierte Variable, wird ein Variablenfehler gemeldet.
  • Syntaxfehler: Wenn der Code nicht der Syntax der Sprache entspricht, wie z.B. fehlende Klammern oder Semikolons, wird ein Syntaxfehler gemeldet.
  • Semantikfehler: Wenn der Code semantisch inkorrekt ist, wie z.B. eine ungültige Verwendung von Operatoren oder eine ungültige Verwendung von Schlüsselwörtern, wird ein Semantikfehler gemeldet.
  • Linting-Fehler: TypeScript enthält auch ein Linting-System, das Entwickler dabei unterstützt, gute Programmierpraktiken zu befolgen und Code-Smell zu vermeiden.
    Ich verwende in dem Modul Integration Architectures bspw. ES-Lint.
  • Readability: Verbesserung der Verständlichkeit bzw. Lesbarkeit des Codes durch explizite Angabe der Typen, sowie Überprüfung der Signaturen von Funktionen und Methoden
