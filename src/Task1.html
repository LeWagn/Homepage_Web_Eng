Aufgabe 1.1.

    1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?

    TBLs Proposal wurde vor allem mit dem Verweis auf mögliche Broken Links abgelehnt. Es wurde als Rückschritt angesehen. Weiterhin wurden die nicht vorhandene Zentrale Kontrolle und die strikte Verwendung von Text Protokollen statt verteilten Objekten genannt.

    2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

    Hauptsächlich durch das Ziel eine Wissensdatenbank und einen Wissensaustausch für das Gemeinwohl bereitzustellen sowie die Einbindung der Community in die Weiterentwicklung der Infrastruktur.

    3. Was wäre der Preis für die garantierte Verhinderung von "broken links"?

    Der Preis dafür keine "broken links", oder anders gesagt referentielle Integrität zu haben, wäre die Prüfung von jedem einzelnen Link, was sehr viele Rechenressourcen kosten würde.

Aufgabe 1.2. HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

    2. Der http Status Code 200 steht für "OK", zugehörig zur Gruppe "Success", er informiert darüber, dass die Anfrage erfolgreich war.

  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

    3. Status Code 301 steht für "Moved permanently", zugehörig zur Gruppe "Redirect" zeigt er an, dass die angefragte Ressource verschoben wurde. Die neue URL sollte im Location Field bereitgestellt werden.

  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

    4. Http Code 400 steht für "Bad Request" und zeigt and, dass der Server die Anfrage nicht bearbeiten kann oder wird. Status Code 400 ist zugehörig zur Gruppe "Client Error" und liegt häufig darin begründet, dass die Anfragesyntax fehlerhaft ist oder ein irreführendes Routing der Anfrage vorliegt.
    Als Nutzer sollte ich meine Eingabe überprüfen, die Seite neu laden und meinen DNS Cache bereinigen.

  4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

    5. Der Status Code 403 steht für "Forbidden", ebenfalls zugehörig zur Kategorie "Client Error" zeigt er an, dass der Server die Anfrage versteht, sich jedoch weigert diese zu autorisieren.
    Häufige Ursachen für Code 403 sind fehlerhafte Security Plugins, korrupte .htaccess Dateien, oder fehlerhafte Zugriffsrechte auf dem angefragten Server.
    Lösungsansätze für Endnutzer sind ähnlich wie die o.g. Ansätze für Code 400.

  5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Aufgabe 1.3. HTML-Literatur

  1. Was ist HTML?
    2.

    Hyper Text Markup Language ist eine textbasierte Auszeichnungssprache, die zur Strukturierung von Dokumenten (Texten, Hyperlinks, Bildern, ...) genutzt wird.
    HTML wurde von TBL entwickelt und wird heute vom W3C und der WHATWG weiterentwickelt. Die aktuelle Version von HTML ist HTML5.
    Obwohl Dokumente mit HTML strukturiert, angepasst und akzentuiert werden können, wird die visuelle Darstellung durch den Webbrowser und Gestaltungsvorlagen wie CSS bestimmt.

  2. Wie kann man eine geschachtelte geordnete Liste der Schachtelungstiefe 3 erzeugen?
    3.

    1. Zuerst öffnet man eine Liste mit den tags für eine geordnete Liste und ein Listen-Item "< ol> < li>"
      1. Statt diese Tags direkt zu schließen, fügt man erneut zwei öffnende tags für eine neue geordnete Liste ein.
        1. Schließlich schachtelt man diese Liste erneut wie zuvor und fügt dann die closing tags pro geöffneter Liste ein, in diesem Fall also jeweils dreimal "< /ol>" und "< /li>"
        2. Durch Hinzufügen von "< li>" tags können beliebig viele Unterpunkte zu den verschiedenen Tiefen der Liste hinzugefügt werden.
  3. Wie ist eine HTML-Tabelle aufgebaut?
    4.

    Eine HTML Tabelle besteht aus Zellen innerhalb von Reihen und Spalten.
    Sie wird mit dem Tag "< table>" erstellt. Neue Zeilen werden mit dem "< tr>" tag eingefügt. Zellen mit dem "< td>" tag (td = table data).
    Überschriften werden mithilfe von sog. Header Zellen ("< th>") erstellt.
    Ein Beispiel für eine HTML Tabelle ist unten bei der Aufgabe 1.4 beigefügt.

  4. Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?
    5.

    Pfade beschreiben den Ort einer Datei innerhalb der Ordnerstruktur der Website. Man unterscheidet zwischen Relativen und Absoluten Pfaden.
    Ein absoluter Pfad ist die volle URL zu einer Datei. Relative Pfade zeigen hingegen auf den Standort der Datei in Relation zum aktuellen Standort. Relative Pfade sollten wenn möglich genutzt werden.
    Als praktisches Beispiel möchte ich hier auf Aufgabe 1.4 verweisen. Das verwendete Bild von TBL befindet sich in einem Unterordner "img", ich rufe es über src = "img/tbl.jpg" ab.
    Ich habe mich also für einen relativen Pfad entschieden, da ich so nicht an die Basis-URL des Bildes gebunden bin.

    In Bezug auf Dateinamen gilt wie üblich in der Programmierung: Vermeidung von Sonderzeichen mit Ausnahme des Unterstrichs "_".
    Dateien sollten weiterhin die Endung .html oder .htm erhalten. Die Länge der Dateinamen sollten je nach Web-Server eine Länge von 256 Zeichen nicht überschreiten.

  5. Wie baut man in HTML ein Menü?
    6.

    Mit purem HTML sollte man generell kein Menü bauen, hier empfiehlt sich eher eine Mischung aus JavaScript, CSS und HTML oder pures CSS.
    Es ist zwar möglich ein Menü nur mit HTML (mithilfe des "< menu>" tags) zu bauen, da dies aber aktuell nur im Firefox Browser unterstützt wird halten sich die Anwendungsfälle in Grenzen.

  6. Welche Attribute sollte man bei Bildern wie verwenden?
    7.

    Bei der Verwendung von Bildern mithilfe des "< img>" tags werden zwei Attribute Benötigt.
    1. src spezifiziert den Pfad zum Ort an dem das Bild gespeichert ist.
    2. alt spezifiziert einen alternativen Text, der angezeigt wird wenn das Bild nicht angezeigt werden kann.
    Als Randnotiz sei gesagt, dass man stets einen alternativen Text für Bilder angeben sollte, damit z.B. Menschen mit eingeschränktem Sehvermögen ebenfalls in der Lage sind den dargestellten Inhalt der Seite wahrzunehmen. (z.B. in Audio Form).
    Weitere optionale Attribute sind z.B. "height" welches die Höhe des Bildes in Pixeln angibt, width (selbsterklärend) und "loading", welches festlegt ob der Browser das Bild unmittelbar oder erst bei Erfüllung einer bestimmten Bedingung laden soll.
